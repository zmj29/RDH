## 2. 项目基本信息

项目名称：

`CNN-Prediction-Based-Reversible-Data-Hiding`

对应论文：

Runwen Hu, Shijun Xiang. *CNN Prediction Based Reversible Data Hiding*. IEEE Signal Processing Letters, 2021.

项目类型：

- 这是一个**基于已有论文方法的复现与实现型项目**
- 重点在于**加载预训练 CNN 预测器并完成嵌入端实验**
- 不是一个从零训练模型、从零搭建完整实验平台的大型工程

一句话概括：

这是一个利用 CNN 预测器提升像素预测精度，并结合扩展嵌入和直方图平移两种可逆信息隐藏方式，对灰度图像进行数据嵌入并评估图像质量的项目。

## 3. 项目要解决的问题

可逆信息隐藏（RDH）的核心要求是：在图像中嵌入秘密信息后，既要尽量减小图像失真，又要能够在理论上恢复出原图和原始消息。

这篇论文和该项目的主要关注点是：

- 改进图像像素预测的准确性；
- 让预测误差更加集中；
- 在相同载荷条件下提高含密图像的保真度；
- 通过更好的预测器，为传统 RDH 嵌入方法提供更好的预测基础。

项目的思路不是发明一种全新的嵌入公式，而是用一个轻量级 CNN 预测器替代传统线性预测器，再把这个预测器接到已有的可逆信息隐藏框架中。

## 4. 项目已经实现的内容

当前项目**已经实现**了以下部分：

### 4.1 CNN 预测器

项目中实现了一个轻量级 CNNP 预测网络，主要特点是：

- 输入为灰度图像上下文；
- 使用 `3×3`、`5×5`、`7×7` 三个感受野分支并行提取特征；
- 再通过后续卷积层对预测结果进行融合与细化；
- 网络规模较小，适合作为预测器使用。

### 4.2 两阶段嵌入流程

项目采用了与论文一致的两阶段处理方式：

- 先将图像按棋盘格划分为两类像素集合；
- 第一阶段使用一类像素预测另一类像素并嵌入一半消息；
- 第二阶段再反过来处理另一类像素并嵌入剩余消息。

从实现角度讲，这相当于：

- 先处理一组像素；
- 再处理互补像素；
- 这样可以减少相邻像素同时修改带来的干扰。

### 4.3 两种嵌入模式

项目中实际实现并跑通了两种嵌入方式：

1. `expansion_embedding`
   即扩展嵌入。

2. `histogram_shifting`
   即直方图平移。

其中：

- 扩展嵌入实现相对直接；
- 直方图平移还引入了背景复杂度排序和阈值估计；
- 从已跑出的结果看，直方图平移模式整体图像质量更好。

### 4.4 推理入口与实验运行

项目提供了可运行的主入口，能够完成：

- 加载预训练模型；
- 启动 MATLAB Engine；
- 读取测试图像；
- 执行嵌入流程；
- 计算含密图像与原图之间的 `PSNR`；
- 输出平均结果。

在后续补充脚本后，还已经实现了：

- 批量导出逐图像结果；
- 导出平均结果表；
- 导出环境说明；
- 导出原始日志；
- 导出图表源数据与图表图片。

## 5. 项目当前没有实现的内容

这是最重要的边界说明。以下内容**不要误写成已完成**：

### 5.1 没有训练代码

虽然原论文说明了 CNNP 的训练方式，但当前项目并没有提供完整训练流程，包括：

- 数据集加载器；
- 训练脚本；
- Adam 优化训练入口；
- loss 计算与训练日志；
- 模型从头训练到收敛的代码链。

因此，当前项目更准确的定位是：

> 使用预训练权重完成推理与实验，而不是完整复现训练过程。

### 5.2 没有统一的提取与恢复入口

当前项目没有提供一套可直接运行的“可逆性闭环验证脚本”，也就是说没有统一入口去完成：

- 消息提取；
- 原图逐像素恢复；
- 提取消息与原消息比对；
- 恢复图像与原图比对。

虽然 MATLAB 嵌入脚本内部保留了恢复所需的部分侧信息思路，但项目中没有现成、可直接运行的恢复验证流程。

### 5.3 没有 BER / 恢复成功率实验

因此，当前项目也**不能直接给出**：

- `BER`
- 消息提取是否正确
- 原图是否逐像素恢复
- 恢复成功率

这些内容如果出现在论文里，只能写成：

- 当前项目尚未完成的实验部分；
- 或后续可继续扩展的工作。

### 5.4 没有传统预测器的完整对比实现

原论文中拿 CNNP 与一些传统预测器做了系统对比，但当前项目代码里并没有完整提供：

- `BIP`
- `MEDP`
- `GAP`
- `DP`

这些传统预测器的完整程序级实现与统一实验平台。

所以当前项目不应被写成“已经完整复现了论文中的全部基线对比实验”。

## 6. 项目的实现结构

从工程结构上看，这个项目是一个 **Python + MATLAB 混合实现**：

### 6.1 Python 负责的部分

- 图像读取与预处理；
- 构造 CNN 输入；
- 调用 CNN 模型进行预测；
- 调用 MATLAB Engine；
- 计算 `PSNR`；
- 批量整理结果与导出 `csv`。

### 6.2 MATLAB 负责的部分

- 扩展嵌入主流程；
- 直方图平移主流程；
- 背景复杂度计算；
- 阈值 `Tp / Tn` 估计；
- 位置图等侧信息处理。

所以这个项目不是纯 Python 项目，而是：

> 以 Python 作为主控，以 MATLAB 实现嵌入算法细节的混合式实验工程。

## 8. 当前已经得到的实验结果

当前项目已经完成的核心实验是：

- `2` 种模式
- `4` 个载荷
- `4` 幅图像

也就是总共：

- `32` 条逐图像记录
- `8` 条平均结果

### 8.1 平均 PSNR 结果

`histogram_shifting` 模式在 `10000 / 20000 / 50000 / 100000 bit` 下的平均 `PSNR` 分别为：

- `55.48`
- `51.79`
- `46.82`
- `42.54`

`expansion_embedding` 模式在相同载荷下的平均 `PSNR` 分别为：

- `47.09`
- `44.62`
- `40.68`
- `37.45`

### 8.2 当前结果支持的结论

基于目前已经跑出的结果，可以安全得出以下结论：

1. 随着嵌入载荷增加，图像质量下降。

2. 在相同载荷下，`histogram_shifting` 的图像质量优于 `expansion_embedding`。

3. 该项目在当前实现下，已经能够稳定支持“不同模式和不同载荷对图像失真的影响分析”。

### 8.3 当前结果不能支持的结论

目前还**不能**据此声称：

- 已完成完整可逆性验证；
- 已证明消息能够无误提取；
- 已证明原图能够逐像素完全恢复；
- 已复现实验论文中的全部大规模对比实验。
