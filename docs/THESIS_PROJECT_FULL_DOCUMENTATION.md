## 3. 项目一句话概括

这是一个基于论文 **CNN Prediction Based Reversible Data Hiding** 的复现型项目。它的核心能力不是训练新模型，而是：

- 加载一个已经训练好的 CNN 预测器；
- 对输入灰度图像执行两类可逆信息隐藏流程；
- 计算含密图像与原图之间的 PSNR；
- 输出整个测试图像集合的平均 PSNR。

## 4. 项目的真实定位

## 10. 采样方式与两阶段处理机制

这是整个项目最关键的设计之一。

### 10.1 `parse_sample()` 做了什么

`utils.parse_sample(img_gray, num)` 的作用不是简单裁剪图像，而是构造一种稀疏输入：

- 创建与原图同尺寸的全零矩阵 `source`
- 只在内部像素区域 `1 ~ H-2, 1 ~ W-2` 上操作
- 当 `(i + j) % 2 == num` 时，认为当前像素属于目标子集
- 把该目标像素的四邻域值复制到 `source` 中对应位置：
  - 上
  - 下
  - 左
  - 右
- 如果 `num == 1`，则把构造好的 `source` 顺时针旋转 90 度

因此，CNN 并不是直接拿原图做输入，而是拿“围绕待预测像素的四邻域稀疏上下文”做输入。

### 10.2 两阶段嵌入

两种嵌入模式都采用相同的两阶段外层框架：

1. 第一阶段处理一半像素子集。
2. 第二阶段处理另一半像素子集。

具体实现为：

- `for i in range(0, 2):`
- 第一轮 `num = 0`
- 第二轮 `num = 1`

这样做本质上对应论文里常见的棋盘格划分思想，即把像素划分为两个互补子集，分两次处理，避免直接相邻像素同时被修改带来干扰。

### 10.3 消息长度如何分配

代码把消息长度一分为二：

```python
half_message = np.int32(len(message_to_embed) / 2)
```

两轮分别嵌入：

- 前一半
- 后一半

需要注意一个实现细节：

- 如果 `watermark-length` 为奇数，`np.int32(len/2)` 会向下取整；
- 这样最后可能会丢掉 1 个比特；
- 因此从代码严谨性角度讲，推荐使用偶数长度。

## 11. CNNP 神经网络结构

### 11.1 总体结构

`model/cnnp.py` 定义了一个非常轻量的多分支卷积预测器：

- `conv1`：`3×3` 感受野分支
- `conv2`：`5×5` 感受野分支
- `conv3`：`7×7` 感受野分支
- `conv4`：对前三个分支的和继续卷积融合
- `conv5`：进一步融合并输出单通道预测结果

每个主分支的通道数都是 `32`。

### 11.2 每个分支的层级

网络各部分可以写成：

- `conv1`
  - `Conv2d(1, 32, 3, stride=1, padding=0)`
  - `LeakyReLU`
  - `Conv2d(32, 32, 3, stride=1, padding=1)`
- `conv2`
  - `Conv2d(1, 32, 5, stride=1, padding=1)`
  - `LeakyReLU`
  - `Conv2d(32, 32, 3, stride=1, padding=1)`
- `conv3`
  - `Conv2d(1, 32, 7, stride=1, padding=2)`
  - `LeakyReLU`
  - `Conv2d(32, 32, 3, stride=1, padding=1)`
- `conv4`
  - `Conv2d(32, 32, 3, stride=1, padding=1)`
  - `LeakyReLU`
  - `Conv2d(32, 32, 3, stride=1, padding=1)`
- `conv5`
  - `Conv2d(32, 32, 3, stride=1, padding=1)`
  - `LeakyReLU`
  - `Conv2d(32, 1, 3, stride=1, padding=1)`

### 11.3 前向传播方式

前向过程是：

1. 三个不同感受野分支并行提取特征
2. 将三者结果相加
3. 经过 `conv4` 细化
4. 再与前三者共同相加
5. 经过 `conv5` 输出预测图

对应公式可写成：

```text
out1 = conv1(x)
out2 = conv2(x)
out3 = conv3(x)
out4 = conv4(out1 + out2 + out3)
out5 = conv5(out1 + out2 + out3 + out4)
```

### 11.4 参数规模

按代码逐层计算，网络总参数量约为：

```text
58,529
```

这说明它是一个非常轻量的预测模型，适合在论文中表述为“轻量级多感受野 CNN 预测器”。

### 11.5 张量尺寸变化

在 `512×512` 输入下：

- 稀疏输入经 `ToTensor()` 和 `unsqueeze(1)` 后变为 `1×1×512×512`
- 三个分支的第一层卷积输出都被设计为 `510×510`

原因如下：

- `3×3, padding=0`：`512 -> 510`
- `5×5, padding=1`：`512 -> 510`
- `7×7, padding=2`：`512 -> 510`

这样三个分支的输出尺寸一致，便于相加融合。

最终：

- 网络输出尺寸为 `1×1×510×510`
- 再被放回到 `512×512` 的大图内部区域
- 边界 1 像素不参与预测

## 13. 扩展嵌入算法 `cnn_expansion.m`

### 13.1 算法核心思想

扩展嵌入模式基于预测误差扩展思想实现。它利用预测值 `xPredict` 与真实像素 `x` 的关系，把误差扩展后用于嵌入比特。

### 13.2 主要变量

- `imgW`
  - 当前水印图像
- `oddOrEvenPlace`
  - 标记本轮处理的像素位置
- `locationMap`
  - 标记不可嵌入位置
- `watermarkLength`
  - 本轮消息长度
- `inserablePlace`
  - 可扫描嵌入的候选位置

### 13.3 不可嵌入位置判定

对候选像素计算：

```text
value = 2*x - xPredict
```

如果：

- `value > 254`
- 或 `value < 0`

则该位置被标记为不可嵌入，记入 `locationMap`。

这一判定的目的，是保证扩展后的像素值不会越界。

### 13.4 位置图压缩与 LSB 预留

代码会先计算位置图 `locationMap`，再用 `arithenco` 做算术编码压缩。

随后，程序会：

1. 从图像末端提取一部分 LSB。
2. 把这些 LSB 与待嵌入消息拼接。
3. 把位置图长度、位置图内容、消息长度、`number1`、`number0` 等元数据写回图像 LSB。

这里用到：

- `mn = nextpow2(M*N)`

对于 `512×512` 图像：

- `M*N = 262144`
- `mn = 18`

所以扩展嵌入中保留的固定元数据长度为：

```text
4 * mn = 72 bit
```

### 13.5 真正的比特嵌入公式

定义预测误差：

```text
dij = x - xPredict
```

若待嵌入比特为 `b`，则：

```text
Dij = 2*dij + b
imgW = xPredict + Dij
```

也就是说，扩展嵌入是把原有预测误差乘 2，再把比特塞进最低位。

### 13.6 算法特点

- 结构相对直接
- 不需要额外计算复杂度排序
- 主要依赖位置图保证可逆性
- 失真通常比直方图平移更大一些

## 14. 直方图平移算法 `cnn_histogram_shifting.m`

### 14.1 算法核心思想

该模式同样基于预测误差，但额外引入了：

- 背景复杂度排序
- 误差直方图阈值选择

它的目标是优先在更平滑的区域嵌入消息，以尽量降低失真。

### 14.2 背景复杂度的定义

`calculate_complexity.m` 中的复杂度计算为：

```text
C(i,j) = (|x(i,j-1)-x(i-1,j)| + |x(i-1,j)-x(i,j+1)|
        + |x(i,j+1)-x(i+1,j)| + |x(i+1,j)-x(i,j-1)|) / 4
```

这个值越小，通常表示局部区域越平滑。

### 14.3 排序策略

代码先对本轮候选像素计算复杂度，再按复杂度升序排序：

```text
[~, indexComplexity] = sort(complexity(inserablePlace))
```

这意味着：

- 真正用于载荷嵌入的位置优先来自低复杂度区域
- 用于 LSB 侧信息替换的位置来自高复杂度区域尾部

这是该实现中非常关键的细节。

### 14.4 误差直方图阈值 `Tp` 与 `Tn`

`calculate_tp_tn.m` 的做法是：

1. 对预测误差做直方图统计
2. 每次取当前频数最大的误差值
3. 累加，直到累计频数达到消息长度
4. 取这些误差值中的最大值为 `Tp`
5. 取最小值为 `Tn`

可以理解为：

> 选出一段最能承载当前消息长度的高频误差区间。

### 14.5 LSB 元数据长度

直方图平移中需要保存的固定元数据更多，包括：

- 压缩位置图长度
- 消息长度
- `Tp`
- `Tn`
- `number1`
- `number0`

因此固定元数据长度为：

```text
6 * mn
```

对 `512×512` 图像来说即：

```text
108 bit
```

### 14.6 真正的嵌入规则

仍定义预测误差：

```text
dij = x - xPredict
```

若误差落在阈值区间外，则先做平移：

```text
if dij > Tp:
    Dij = dij + Tp + 1
elif dij < Tn:
    Dij = dij + Tn
```

若误差落在可嵌入区间内，则嵌入比特 `b`：

```text
Dij = 2*dij + b
imgW = xPredict + Dij
```

### 14.7 算法特点

- 需要额外计算复杂度与阈值
- 逻辑比扩展嵌入更复杂
- 在现有结果中 PSNR 整体优于扩展嵌入

## 15. 两种模式的共同点与差异

### 15.1 共同点

- 都是两阶段处理
- 都依赖 CNN 预测值
- 都使用位置图保证可逆性
- 都先做部分 LSB 替换来保存侧信息
- 都最终输出含密灰度图

### 15.2 差异

扩展嵌入：

- 直接对预测误差做扩展
- 没有复杂度排序
- 固定元数据更少
- 实现更直接

直方图平移：

- 先做复杂度排序
- 再估计 `Tp`、`Tn`
- 元数据更多
- 算法更复杂
- 当前结果中图像质量更好
